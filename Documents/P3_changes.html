Endringer på styling

  • Font: "Helvetica, Arial, sans-serif" for både header, body og footer
  • Color scheme: Oransje farge sammen med mørke/sorte farger for nettsiden generelt
  • Body width: 60% + 100px. Minste bredde for hele siden er 525px
  • Header: Headeren vil bestå av det fulle navnet til laget, altså Det Frie Student Gamer Laget, samt en logo på venstre siden
  • Footer: Footer vil innholde kontaktinfo, mer info om laget samt et kart over lokasjonen til laget
  • Background Image: Gaming relatert bilde som dekker siden

Javascript krav 1: list_item.js

Timmy har oppfylt JavaScript-kravet med å dynamisk laste inn elementer i en liste.

Javascript krav 2: leetspeak.js

Håvard har oppfylt JavaScript-kravet med å implementere en easter egg på siden som konverterer noe av teksten til Leetspeak. Den kan testes ved å klikke på "Bli medlem!"-knappen øverst til høyre.

Javascript krav 3: slideshow.js

Timmy har oppfylt JavaScript-kravet med å implementere en dynamisk slideshow for å vise både bilder og artikeltekst. Det er brukt JavaScript sammen med CSS-transitions for å oppnå dette.

Javascript krav 4: joinuspopup.js

Sandor har oppfylt JavaScript-kravet med å implementere en enkel popup som vises for nye besøkende for å appellere dem til å bli medlem. Implementasjonen bruker localStorage så for den må cleares for testing.

Modulæritets krav:

Vi har brukt en kombinasjon av Apache SSI og JavaScript for å oppfylle modulæritets-kravet. Vi bruker SSI for å laste inn meta.html, header.html, footer.html og joinuspopup.html HTML templatene på alle sidene. Webserveren siden kjører på er derfor avhengig av å kjøre Apache og hvor SSI modulen er slått på. Apache instillingene settes så i .htaccess filen. Sandor stod for proof of concept med Apache og selve implementasjonen av config, men templatene som ble brukt ble skrevet av alle.

JavaScript brukes så for å laste inn lage slideshow og list-items. Timmy stod for implementasjonen.

Arbeidsoppgaver

Listen under viser minimmumskravene til enkeltpersonene med oppdatert informasjon om de ble utført eller ikke. Listen inneholder også ny arbeidsoppgaver som ble lagt til under utviklingen av siden.

Uthevet skrift representerer oppdatering, mens skrift som er strøket over representerer gammel/endret informasjon.
